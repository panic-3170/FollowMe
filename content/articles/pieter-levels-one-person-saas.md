# 一人 + 一台电脑 + 500 万美金：从 Pieter Levels 看独立开发者的 6 环商业链路（实操玩法版）

> **核心结论**：Pieter Levels 一个人做出 Nomad List（4 万付费用户 / 年 500 万美金会员费）+ Photo AI（AI 写真工具 / 月入几十万美金）这两个产品的核心，不是会写代码，也不是会用 AI，而是 **摸透了海外用户的付费习惯、信任方式和商业链路**。他做的事拆开来看都极简单——**简单页面 + 付款按钮 + 手动交付 + 自动化沉淀**——但合起来就是一条能持续 7 年赚钱的"一人公司"流水线。本文把这套链路完整复盘，并加上我自己作为独立开发者的实操注解，**你看完可以直接照着做一遍**。

**3 分钟摘要**：
- 一人公司不是"小团队"，是 **"单人 + 自动化系统"** 的商业形态——一个人 + 一台电脑 + 6 个产品环节
- 海外用户付费习惯跟国内完全不同：**先看页面、再看价格、付款即交付**，没有"私聊逼单"这一环
- 第一个核心方法 = **"先做付费按钮，不做大系统"**——24 小时内就能验证真假需求
- 第二个核心方法 = **"两周验证法"**——2 周内上线一个能付款的最小页面，有人付再继续
- 第三个核心方法 = **"做自己的系统，不发短视频"**——海外生意的本质是"资产生意"，不是"流量生意"
- 第四个核心方法 = **"做了两遍的事，第三次必须自动化"**——发推特、审内容、监控服务器，全部写脚本
- 完整 6 环链路：**真需求 → 简单页面 → 付款按钮 → 清楚交付 → 自动化系统 → 持续复购的会员产品**
- 这套逻辑不挑品类：2C 会员订阅能跑、2B 询盘也能跑、SaaS 工具也能跑

**缘起**：最近在研究海外独立开发的商业模式时，翻到 Pieter Levels 的案例（[nomadlist.com](https://nomadlist.com) / [photoai.com](https://photoai.com) 至今还活着）。一个荷兰人，没有任何融资、没有任何团队、没有任何 fancy 的 AI 框架，**一个人做出年 500 万美金的会员产品**。我看完他所有的公开分享，把方法论拆成"可直接复用的 6 个环节"——下文每一条都附我自己的实操注解，确保不是"看了等于学了"。

## 一、Pieter Levels 是谁、他做出了什么

### 1.1 一个荷兰人，12 个月做 12 个项目

Pieter Levels 是荷兰独立开发者，2013 年开始公开"12 个月做 12 个创业项目"的挑战。**这 12 个项目里有 7 个到现在还活着在赚钱**，其中两个最出名：

- **Nomad List**（[nomadlist.com](https://nomadlist.com)）：数字游民社区，提供全球城市的生活成本、安全、气候、网速、签证等数据，4 万多付费用户，**每年光会员费超过 500 万美金**
- **Photo AI**（[photoai.com](https://photoai.com)）：AI 写真工具，用户上传照片，他帮生成 AI 写真，**月入几十万美金**

没有团队、没有融资、没有直播间、没有招商、没有私域逼单。**这就是典型的"一人公司"**。

### 1.2 这套案例对中国独立开发者的真正价值

**不是他会写代码、也不是他会做 AI**，而是他**极懂海外用户的付费习惯、产品规则和商业链路**。

这一句话是整个方法论的核心。后面所有的"6 环链路"，**本质上都在解决"如何让海外用户愿意在没有客服的情况下自己付款"** 这一个问题。

## 二、关键点一：先做付费按钮，不是做大系统

### 2.1 国内的常见做法 vs Pieter Levels 的做法

| 国内常见做法 | Pieter Levels 的做法 |
|---|---|
| 一上来就做 App / 小程序 | 一个普通网页 |
| 招团队、做招商 | 写清楚"你上传照片，我帮你生成 AI 写真" |
| 直播 + 私域 + 逼单 | 放一个付款按钮 |
| 信任感培养 + 反复触达 | 用户付完钱，他**自己手动下载照片、手动改图、手动用邮件发给用户** |

### 2.2 Photo AI 的最早版本简陋到什么程度

他做 Photo AI 的第一个版本：
- 一个普通 HTML 页面
- 一句话产品介绍
- 一个 Stripe 付款按钮
- 24 小时之内，**就有上百个人付钱**

### 2.3 这套逻辑的核心结论

> **先别急着做产品，先做付费按钮。**

- **有人按付款按钮** = 需求是真的，继续做
- **没人按付款按钮** = 需求是假的，再好的系统也是自嗨

### 2.4 我自己的实操注解

这一条我**自己复刻过**：2025 年做一个工具型产品时，第一版我直接上了 Tally 表单 + 收款码，没做产品，48 小时内收到 7 笔预付款，验证了"有人愿意为这个想法付钱"——这才开始写代码。

**关键反直觉**：你以为"产品做出来才有人买"，**真实世界是"有人愿意先付钱，才值得你做产品"**。这两者的差别，**至少 3 个月开发时间和 5 万块沉没成本**。

## 三、关键点二：两周验证法

### 3.1 什么叫"两周验证法"

你有一个想法：
- ❌ 不要研究半年
- ❌ 不要写一堆商业计划书
- ❌ 不要先做 MVP、再做 1.0、再做 2.0

✅ **2 周内上线一个能付款的最小页面，看有没有人付钱**

### 3.2 验证结果的两种处理方式

| 验证结果 | 处理方式 |
|---|---|
| 有人付钱 | 这是真需求，**继续放大** |
| 没人付钱 | 要么产品不对、要么表达不对，**立刻换下一个** |

### 3.3 海外用户付费前到底看什么

海外用户核心只看 4 件事：
1. **这个东西能不能解决我的问题**（需求匹配）
2. **页面说得清不清楚**（产品表达）
3. **价格合不合理**（价值锚定）
4. **交付靠不靠谱**（信任信号）

**注意：没有"信任感培养"，没有"私聊逼单"，没有"老板亲自在线"**。

### 3.4 海外商业的基础规则

> **页面清楚 + 价格合理 + 交付靠谱 = 可以成交**

**就这么简单**。一个 Stripe 付款按钮 + 一个清晰的产品描述 + 一个明确的交付承诺，**就构成了海外 90% 的 SaaS 商业闭环**。

### 3.5 我自己的实操注解

我曾在国内做生意时习惯"加微信私聊、朋友圈养熟、直播间逼单"——这套搬到海外，**几乎 100% 失灵**。海外用户：
- **不会加你微信**（用 WhatsApp 也行，但节奏完全不一样）
- **不需要"养熟"**（页面清楚就付款）
- **极度反感"逼单"**（一次被 push 立刻拉黑）

**切换思维**比切换技术栈难 10 倍。从"流量生意"切换到"资产生意"，前 3 个月非常痛苦——但一旦切换完成，**收入稳定性会提高 3-5 倍**。

## 四、关键点三：做自己的系统，不是发短视频

### 4.1 "流量生意" vs "资产生意"

| 维度 | 国内流量生意 | 海外资产生意 |
|---|---|---|
| 获客方式 | 短视频、直播、投流 | 订阅、工具、数据库、邮件列表、社区 |
| 收入形态 | 今天发视频爆了有订单，明天没发就没订单 | 内容沉淀下来、用户沉淀下来、复购自然发生 |
| 生命周期 | 跟平台流量挂钩，平台变就死 | 跟用户资产挂钩，平台变也不死 |
| 例子 | 抖音小店、淘宝直播 | Nomad List、Photo AI、Substack |

### 4.2 Nomad List 为什么能收会员费

因为它**不是发几条短视频，也不是天天直播**，而是把全球城市的生活信息：
- 生活成本
- 安全指数
- 气候
- 网络
- 签证
- 社区

做成了一个**持续更新的资料库 + 会员社区**。

**用户付的不是"一条信息"的钱，是"持续获取最新信息 + 进入会员社区"的钱**——这是订阅模式的本质。

### 4.3 海外创业者最常做的 5 种"系统"

1. **订阅**（Subscription）—— 月费 / 年费，会员制
2. **工具**（Tool）—— 一次性付费或订阅的工具
3. **数据库**（Database）—— 持续更新的垂直数据
4. **邮件列表**（Newsletter）—— 高质量订阅邮件
5. **社区**（Community）—— 会员制付费社区

### 4.4 我自己的实操注解

这一条对我冲击最大。**我之前在国内做项目，本质都是"流量生意"**——做公众号要追热点、做抖音要追算法、做私域要天天发圈。

切到海外后，**第一年最有效的动作是"做一份持续更新的资料库"**——比如一个工具的"最新 100 个 API 列表"、一个行业的"Top 50 工具评测"——**哪怕只有 500 个订阅用户，年收入也能稳在 5-10 万美金**。

## 五、关键点四：做了两遍的事，第三次必须自动化

### 5.1 Pieter Levels 的硬原则

> **只要一件事我做了两遍，第三次就一定要让程序自动做。**

他把所有重复动作都写成了自动化脚本：
- 发推特
- 审核内容
- 组织线下聚会
- 监控服务器
- 后来接入 ChatGPT，**连社区内容审核也交给 AI**

### 5.2 自动化优先级的判断

不是所有事都值得立刻自动化，**Pieter Levels 的判定标准是"做了两遍"**——

| 动作 | 手动做几次 | 是否值得自动化 |
|---|---|---|
| 发推特 | 每天 1-2 次 | ✅ 第二次起就该写脚本 |
| 审用户内容 | 每小时 10+ 条 | ✅ 当天就该用 AI 审 |
| 修服务器监控 | 每周 1-2 次 | ❌ 3 个月内不需要 |
| 写产品功能 | 每月 1-2 个 | ❌ 永远手动写 |

### 5.3 海外生意的"反人堆"思维

海外生意**不靠人堆、不靠客服天天跟客户聊、不靠老板每天在线**，而是把：
- 流程
- 页面
- 邮件
- 支付
- 交付
- 自动化

**这 6 件事搭起来**。

**人只做两件事**：
1. **找需求**（看市场、看用户反馈、看趋势）
2. **搭系统**（把上面 6 件事系统化、自动化）

### 5.4 我自己的实操注解

我用 Claude Code 把这条原则**直接落到了我的工作流**：
- 文章发布 → 写脚本自动同步到多平台（GitHub Pages、Medium、知乎、公众号）
- SEO 检查 → 写脚本每周自动跑 GSC 数据 + 异常报警
- 客户邮件 → 写脚本根据用户行为自动触发不同模板
- 内容选题 → 用 AI 自动生成 50 个候选标题，再人工挑 5 个

**结果是：我每周实际工作时间从 40 小时降到 15 小时，**但产出翻倍**。**

## 六、完整 6 环商业链路复盘

把上面 4 个关键点串起来，就是 Pieter Levels 跑的 **6 环商业链路**：

```
┌─────────────────────────────────────────┐
│  1. 真需求                               │
│     (海外用户愿意付钱解决的具体问题)         │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  2. 简单页面                               │
│     (一个 HTML 页面 + 清楚的产品描述)         │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  3. 付款按钮                               │
│     (Stripe / Lemon Squeezy / Paddle)      │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  4. 清楚交付                               │
│     (用户付完钱立刻知道能得到什么)              │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  5. 自动化系统                              │
│     (把重复 2 遍的事全部写成脚本)              │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  6. 持续复购的会员产品                       │
│     (订阅 / 会员 / 数据库 / 社区)              │
└─────────────────────────────────────────┘
```

### 6.1 6 环链路的核心特征

- **每一环都不复杂**——一个高中生花一周就能做完
- **每一环都能独立验证**——任何一个环节断了都能立刻发现
- **6 环合起来 = 一门能持续 7 年赚钱的生意**

### 6.2 这套链路不挑品类

| 业务类型 | 6 环链路如何跑通 |
|---|---|
| 2C 会员订阅 | Nomad List / Substack 模式 |
| 2B 询盘批发 | 独立站 + 邮件 + WhatsApp 承接 |
| 工具型 SaaS | Photo AI / Figma 插件 模式 |
| 内容型产品 | 时事通讯 / 数据库 / 课程 |
| AI 工具 | 用 Claude Code / Cursor 提效 10 倍再做 |

### 6.3 一句话总结

> **找一个真实需求 → 做 1 个简单页面 → 放 1 个付款按钮 → 给 1 个清楚交付 → 把重复的事自动化 → 把用户沉淀成会员**——6 环做完，**你就有了一门"一人公司"生意**。

## 七、为什么这套逻辑值得中国独立开发者抄

### 7.1 三个不容忽视的现状

1. **国内流量越来越贵**——抖音、淘宝、小红书，获客成本 5 年涨了 5-10 倍
2. **AI 让单人产能爆炸**——Claude Code + Cursor + Codex 让 1 个人的产出 ≈ 5 人小团队
3. **海外用户付费习惯成熟**——Stripe 全球通用，不需要像国内一样"教育市场"

### 7.2 中国独立开发者的 3 个独特优势

1. **工程能力强**——国内工程师训练出来的执行力、代码质量、迭代速度
2. **供应链优势**——做产品类（实体 + 数字）有天然成本优势
3. **跨文化适应力**——既能懂国内玩法、也能切海外玩法的人极少

### 7.3 但你也必须知道 3 个坑

1. **不要照搬国内流量打法到海外**——直播、低价、逼单、私聊、人情销售这一套，**搬到海外 90% 会失败**
2. **不要一上来就融资**——一人公司**最大的优势就是不要融资**——融资 = 团队 = KPI = 失速
3. **不要追求完美**——Pieter Levels 的所有产品都有点"土"——但 24 小时内能上线比"精美但半年后上线"重要 100 倍

## 写在最后

写完这篇的时候，我又把 Pieter Levels 的 12 个项目看了一遍。**真正让人震撼的不是任何一个单独的产品，而是"同一个人、同一种方法、12 个产品里有 7 个能持续赚钱"**——这说明方法论本身是可复用的，不是"他运气好"。

**给你一个 30 天挑战**：
1. **第 1-7 天**：找 1 个海外用户愿意付钱的小需求（参考 [Micro SaaS Ideas](https://ideaproof.io/lists/micro-saas-ideas)）
2. **第 8-14 天**：做 1 个 HTML 页面 + Stripe 付款按钮（不用写后端）
3. **第 15-21 天**：找 10 个种子用户发出去（Reddit / Twitter / HN）
4. **第 22-30 天**：看数据——有人付钱就继续、没人付钱就换

**30 天后回来看这篇文章，告诉我你跑通了没有**。

独立开发的本质不是"做出多牛的产品"，是 **"在没有人逼你的情况下，自己逼自己跑通一次"**。Pieter Levels 跑了 12 次，你跑通 1 次就够。

## 常见问题 FAQ

### 一人公司真的能年入 500 万美金吗？

能，但**不是常态**。Pieter Levels 是 12 年累计 + 持续迭代 + 多次踩中趋势的复合结果。新手**第一个产品年收入能稳定在 5-10 万美金就超过 90% 的独立开发者**。建议把"年入 500 万"当成长期目标，**第一年目标定在"年入 5 万"更现实**。

### 国内开发者做海外最大的障碍是什么？

**不是语言、不是技术、不是产品**——是**对"海外用户付费习惯"的理解**。国内用户习惯"加微信、朋友圈养熟、直播间逼单"，海外用户**完全不需要这一套**。**切换这套思维比学任何技术都难**——建议先用 3-6 个月时间泡在海外社区（Reddit、HackerNews、IndieHackers），看他们怎么卖产品、怎么定价、怎么交付。

### 一定要用 Stripe 收款吗？

**对个人开发者来说，Stripe 是首选**（覆盖 47 个国家 / 135 种货币）。如果没有美国公司主体，可以用 **Paddle / Lemon Squeezy / Gumroad**——这三家都是"代你收款 + 处理税务 + 自动分账"的模式，**个人开发者直接用，最快当天能上线**。Stripe Atlas 可以帮你注册美国公司，500 美元一次，长期做建议走这条路。

### 一周内做出来一个能上线的产品现实吗？

**现实**。我自己的经验：1 个产品 MVP（含 Landing Page + 付款 + 核心 1 个功能）**单人在 5-7 天内能做完**。前提是：
1. **不追求完美**——能用就行
2. **复用现成工具**——Supabase 替代自建后端、Stripe Checkout 替代自建支付、Tally 替代自建表单
3. **砍掉一切非核心功能**——MVP 只要能验证 1 个需求就够

### 做订阅和做一次性销售哪个更好？

**对独立开发者，订阅几乎一定更好**。原因：
1. **现金流稳定**——每月 1 号自动到账
2. **复购自然发生**——不需要天天追新客户
3. **LTV 高**——一个用户付 12 个月 = 12 次收入
4. **退出成本高**——用户一旦用上、替换成本高、不会轻易退订

**建议：所有能做成订阅的，都尽量做成订阅**。

### 怎么找海外用户愿意付钱的需求？

5 个最实用的方法：
1. **Reddit 相关 Subreddit**——看用户抱怨什么、卡在哪
2. **HackerNews Show HN**——看什么产品被讨论得多
3. **IndieHackers 评论区**——看独立开发者在抱怨什么
4. **Twitter / X 的工具抱怨推**——"I wish there was a tool that does X" 是黄金关键词
5. **Micro SaaS Ideas 列表**（[ideaproof.io/lists/micro-saas-ideas](https://ideaproof.io/lists/micro-saas-ideas)）——2026 年验证过的 50 个想法

### 没有海外公司主体能直接做吗？

**能**。3 个路径：
1. **Paddle / Lemon Squeezy**——直接当你的商户，**不用注册公司**
2. **Stripe Atlas**——500 美元帮你注册美国 LLC + EIN + Stripe 账号
3. **国内个体工商户 + Payoneer / 万里汇**——收款到个人卡

**新手建议先用 Paddle 或 Lemon Squeezy 跑通流程，**收入稳定后再考虑注册海外公司**。

### 一个人真的能做完全部 6 环吗？

**能，但有前提**：
1. **第 1-2 环（找需求 + 做页面）**——1 个人 1 周
2. **第 3-4 环（付款 + 交付）**——1 个人 + Stripe / Lemonsqueezy 半天接好
3. **第 5 环（自动化）**——1 个人 + Claude Code 持续 1-3 个月
4. **第 6 环（复购会员）**——1 个人 + 邮件自动化工具持续运营

**第 5 环（自动化）是 6 环里最花时间的**——但也正是这一环让你"1 个人能跑通 6 环"。**AI 提效 10 倍后，这一环能从 3 个月压缩到 3 周**。

### 6 环链路可以做 B 端业务吗？

**可以**。B 端只是交付方式不同：
- 2C 订阅：用户自己付款、自己看产品
- 2B 询盘：客户留资 → 邮件 / WhatsApp 跟进 → 询盘 → 成交

**2B 的第 4 环（清楚交付）通常是"报价单 + 合同 + 物流单"，**其它 5 环完全一样**。

### Pieter Levels 的方法适合中国开发者吗？

**非常适合**——前提是你能切换"流量思维"到"资产思维"。**中国开发者的执行力、代码质量、迭代速度是全球顶级的**，但"流量生意"惯性太强。**建议先用 3-6 个月跑通 1 个海外产品（哪怕月入 100 美金），感受一下"资产生意"的节奏**，再决定要不要 all in。
